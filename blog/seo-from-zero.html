獨立開發者 SEO 實戰：28 天從 0 到 20 個 Google 點擊

2026 年 5 月 4 日 約 11 分鐘閱讀 作者：Hao0321 Studio

前陣子收到 Google Search Console 寄來的「恭喜你在 28 天內獲得 20 次點擊」通知。對於一個沒有任何外部推廣、純靠內容自然成長的個人站，從 0 到 20 個有機點擊的這段過程其實踩了不少坑。這篇把整個 SEO 冷啟動流程攤開講 — 哪些技術 SEO 必做、哪些是過度優化、為什麼新站前 30 天最關鍵。

起點：完全 0 流量

Hao0321 Studio 官網上線時的狀態：

  • 新註冊網域（過去無歷史）
  • 0 反向連結
  • 未提交 sitemap
  • Google 完全不知道這個網站存在

第一週的 Search Console 數據：曝光 0、點擊 0、平均排名 N/A。這是合理的 — Google 還沒爬過任何頁面。

第 1 週：技術 SEO 必做清單

在等 Google 爬蟲發現你之前，先把基礎打好。我跑的 checklist：

1. 提交 sitemap.xml

手寫一個 sitemap.xml 列出所有頁面，放在 https://hao0321.com/sitemap.xml，然後到 Search Console → Sitemaps 提交。

<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>https://hao0321.com/</loc>
    <lastmod>2026-04-21</lastmod>
    <changefreq>weekly</changefreq>
    <priority>1.0</priority>
  </url>
  <url>
    <loc>https://hao0321.com/blog/</loc>
    <lastmod>2026-04-21</lastmod>
    <changefreq>weekly</changefreq>
    <priority>0.8</priority>
  </url>
  ...
</urlset>

2. robots.txt 別擋到自己

很常見的錯誤：誤把 Disallow: / 留下，導致 Google 完全爬不到。我的 robots.txt 永遠只有：

User-agent: *
Allow: /

Sitemap: https://hao0321.com/sitemap.xml

3. 每個頁面 Meta tags 完整

  • <title>：每頁獨立的 title，60 字元內
  • <meta name="description">：150–160 字元的描述
  • <link rel="canonical">：標準 URL，避免重複內容
  • <meta property="og:*">：Open Graph，分享到 Facebook/Twitter 才有縮圖

4. 結構化資料（Structured Data）

在文章頁加 JSON-LD：

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "...",
  "author": { "@type": "Person", "name": "Hao0321 Studio" },
  "datePublished": "2026-04-21",
  "image": "https://hao0321.com/og-image.jpg"
}
</script>

結構化資料不直接影響排名，但會讓 Google 在搜尋結果顯示「rich result」（如評分星星、發佈日期），點擊率會明顯提升。

5. 速度優化

  • 圖片用 WebP（比 PNG 小 30%）
  • Lighthouse 跑分 95+
  • 初次 contentful paint < 1.5 秒

第 2-3 週：Google 開始發現

提交 sitemap 後大約 5–7 天，第一批頁面被索引。Search Console 開始有數據：

週數曝光點擊平均位置
第 1 週00—
第 2 週40145.2
第 3 週180632.1
第 4 週4101328.5

注意「平均位置」從 45 慢慢降到 28 — 這是 Google 在學習你的內容該排在哪。前 30 天的趨勢比絕對數字重要。

什麼樣的關鍵字會帶流量

把 Search Console 的「查詢」報表打開，前 5 個帶流量的關鍵字：

關鍵字曝光點擊
cloudflare workers d1 範例1205
cloudflare pages 部署 教學853
react babel standalone 瀏覽器604

jwt hmac sha256 worker302

共同點：長尾、技術性、解決具體問題的搜尋。沒有人會搜「Hao0321 Studio」（沒人知道我），但會搜「Cloudflare Workers 怎麼用 D1」這類問題。

內容策略 vs 技術 SEO

實戰下來，內容比技術重要 10 倍。技術 SEO 是「讓 Google 能找到你」，內容是「讓 Google 願意推薦你」。

有效的內容套路

  1. 具體問題 + 具體解法：如「Cloudflare Pages 部署到 custom domain 的踩坑」比「Cloudflare 教學」好 10 倍
  2. 真實數據：「我的網站每月成本 26 元」比「便宜的方案」搜得到
  3. 程式碼 + 截圖 + 表格：Google 喜歡多模態內容
  4. 更新時間誠實：寫「2026 年 4 月實測」比「最新教學」好
  5. 長度足夠：800 字以上是底線，1500–2500 字是甜蜜點

無效的內容套路

  • 關鍵字塞滿：Google 抓得到，反而扣分
  • 用 AI 生整篇：缺乏個人經驗，沒有競爭力
  • 抄別人的整理：Google 會偵測 duplicate content
  • 純展示性的作品集：沒搜尋意圖匹配，難帶有機流量

反向連結（Backlinks）的角色

新站冷啟動最容易卡的就是「沒人連你」。我做了三件事：

  1. 各社群平台簡介加連結：YouTube 頻道、Gumroad 商店、社群 bio。這些是 nofollow，但能引流。
  2. 在相關社群分享文章：技術社團、PTT、Reddit r/SideProject。注意不能 spam，要實質貢獻。
  3. 把作品上 itch.io / Gumroad 並 link 回主站：這些是高權重 dofollow 連結。

結果：第 4 週時有 2 個自然反向連結（一個是別人引用我的 Cloudflare 文章），平均排名再往前 5 位。

過度優化的陷阱

看了太多 SEO 教學會犯這些錯：

  • 每篇都塞 30 個關鍵字到 meta keywords：Google 早就忽略這個欄位
  • 每張圖都加 alt 為 keyword stuffing：alt 應該描述圖片內容，不是關鍵字
  • 內鏈密度過高：同一篇文章反覆連到首頁，反而看起來像 spam
  • 追求 100 分 Lighthouse：95 分跟 100 分對 SEO 沒差，後 5 分往往要犧牲使用者體驗

30 天後的策略

越過冷啟動門檻後，重點轉移到：

  • 內容深度 > 內容廣度：把表現好的文章再深耕，比寫新文章 ROI 高
  • 內鏈生態：把相關文章互相 link，形成內容網絡
  • 更新舊文章：Google 會重新爬取「最近更新」的頁面，給排名加分
  • 季度復盤：每 3 個月看一次 Search Console，砍掉沒帶流量的內容

結語：SEO 是長期遊戲

新站的前 30 天像種子發芽 — 大部分時間什麼都看不到，但底層的根系正在長。技術 SEO 把基礎打好之後，剩下的就是寫真誠的、解決真實問題的內容。20 個點擊雖小，但比起 0 是質變的開始。
